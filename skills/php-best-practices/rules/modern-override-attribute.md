# Override Attribute

## Why it matters
A misspelled method name in a subclass silently creates a new method instead of overriding the parent. The bug only surfaces at runtime when the override is not called. `#[\Override]` turns that silent mistake into a hard error at class-load time — it is the cheapest possible refactor safety net.

## Rule
Add `#[\Override]` to every method that overrides a parent class method or implements an interface method. Without exception.

## Bad
```php
<?php

declare(strict_types=1);

abstract class BaseRepository
{
    abstract public function findById(int $id): ?object;

    public function save(object $entity): void
    {
        // persist logic
    }
}

class UserRepository extends BaseRepository
{
    public function findByid(int $id): ?object  // typo: 'id' lowercase
    {
        // This is a NEW method, not an override — findById is never called
        return null;
    }

    public function sav(object $entity): void  // typo: missing 'e'
    {
        // Again, a brand-new method — parent save() still runs unchanged
    }
}
```

## Better
```php
<?php

declare(strict_types=1);

abstract class BaseRepository
{
    abstract public function findById(int $id): ?object;
}

class UserRepository extends BaseRepository
{
    #[\Override]
    public function findById(int $id): ?object  // PHP errors if parent has no matching method
    {
        return null;
    }
}
```

## Best
```php
<?php

declare(strict_types=1);

interface Renderable
{
    public function render(): string;
}

abstract class BaseWidget implements Renderable
{
    public function render(): string
    {
        return '';
    }

    public function refresh(): void {}
}

final class DashboardWidget extends BaseWidget
{
    #[\Override]
    public function render(): string  // covers interface + parent
    {
        return '<div>dashboard</div>';
    }

    #[\Override]
    public function refresh(): void  // catches parent renames at load time
    {
        // refresh logic
    }
}
```

## Exceptions / trade-offs
Test doubles and mocks generated by PHPUnit or Mockery may not carry `#[\Override]` — this is acceptable because the framework generates those classes outside your control. Do not add `#[\Override]` to methods that intentionally introduce new behaviour in a subclass rather than replacing parent behaviour.

## Static-analysis notes
The PHP engine itself validates `#[\Override]` at class-load time — no static analysis tool is required. PHPStan and Psalm also understand the attribute and can flag missing `#[\Override]` annotations in analysis passes. IDEs highlight candidate methods that should carry the attribute.
